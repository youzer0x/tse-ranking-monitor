# grok 変動要因リサーチ・プロンプト雛形

grok に「直近の東証 日中（レギュラー）セッションで、ある1銘柄がなぜ上昇したか」を一次情報で裏取りさせ、Claude が機械的に取り込める **DIGEST_BLOCK** を必ず末尾に出力させるための共有プロンプト。

===== PROMPT START =====

あなたは日本株の適時開示・主要金融メディアに精通したリサーチャーである。下記の1銘柄について、**直近に完了した東証 日中（レギュラー）セッション（{{SESSION_DATE}} 09:00–15:30、前場9:00-11:30／後場12:30-15:30）でなぜ株価が上昇したか（変動要因）**を、一次情報・主要メディアで裏取りして調査せよ。最後に、定型の **DIGEST_BLOCK** を必ず出力すること。

## 対象銘柄（build_day_ranking.py の確定データ）
- コード: {{CODE}}
- 銘柄名（J-Quants 正式名）: {{NAME}}
- 市場: {{MARKET}}
- セッション日: {{SESSION_DATE}}（前営業日: {{PREV_DATE}}）
- 当日 値上がり率: +{{PCT}}%（当日 調整済み終値 ÷ 前営業日 調整済み終値）
- 当日 終値(円): {{CLOSE}} ／ 売買代金(百万円): {{TURNOVER_M}} ／ 時価総額(億円): {{MCAP_OKU}}
- TDnet 厳密窓の開示（前営業日15:30以降 ∪ 当日15:30未満。build_day_ranking.py が取得済み）:
{{DISCLOSURES}}

## 調査方針（厳守）

1. **厳密窓**：日中（9:00–15:30）の上昇の材料になり得るのは、(a) **前営業日の引け後（15:30以降）に出て当日寄りに織り込まれた**事象、(b) **当日 15:30 より前（場中）**に伝わった事象に限る。**当日 15:30 ちょうどの開示は対象外**（後場終了後で日中に反応不能＝今夜の PTS 材料）。直接トリガーの **配信/開示時刻（trigger_time）** を特定し、この窓に整合するか（window_ok）を必ず判定せよ。
2. **優先順位**：**[開示]（上記 TDnet 厳密窓の開示）→[報道]（主要メディアの一次記事。配信時刻を当日セッションに整合確認）→[テーマ]（個別の新規材料が無い場合のみ。半導体/AI・防衛/宇宙・地合い・需給）** の順で特定する。安易に[テーマ]で片付けない。
3. **証券会社のレーティング変更は必ず確認**：投資判断・目標株価の変更は **TDnet に出ない**ため見落としやすい。`https://kabutan.jp/stock/news?code={{CODE}}`（ブラウザ User-Agent）の「レーティング日報」「材料」記事を確認し、結果を `broker_rating` に記す。**前営業日引け後〜当日朝（寄り前）〜当日15:30前**に伝わった格上げ・目標株価引き上げは日中上昇の有力材料。根拠がレーティングのときも区分は **[報道]**（証券会社名・旧→新の投資判断・目標株価を具体的に）。
4. **ソース方針（3層）**：
   - **① 中核 whitelist（常に採用可）**：日本経済新聞・日経CNBC・Bloomberg・Reuters/ロイター・WSJ・Financial Times・CNBC（中核）／時事通信・共同通信・東洋経済オンライン・ダイヤモンド・オンライン・株探ニュース・M&A Online・財経新聞・Nikkei Asia・MarketWatch・Barron's（拡張）／一次情報＝TDnet・企業公式IR/プレスリリース・取引所・TOB公開買付届出。
   - **② 評価採用層（whitelist 外でも、次をすべて満たせば採用してよい。要フラグ）**：(a) 確立した経済・金融報道機関 or 認知された調査会社の **編集記事**であり、(b) 発行・編集の主体が明確で、(c) 検証可能な配信時刻と URL があり一次情報にひも付く。例＝**フィスコ（調査会社の編集記事）・みんかぶの編集ニュース記事・日刊工業新聞等の業界専門紙の編集記事**。**whitelist 媒体が掲載する専門家コラム（例：株探掲載の杉村富生コラム）は当該媒体の編集物として可**。採用したものは `sources_new_candidate` に「媒体名／採用理由／URL」で必ず明記する。
   - **③ ハード除外（絶対に根拠にしない）**：X/Twitter 個人・note.com・独立した個人ブログ/Substack・Reddit/5ch 等の掲示板・YouTube/TikTok 個人・匿名まとめ・**純アルゴ自動生成（例：Simply Wall St）**・みんかぶ等の**ユーザー投稿/個人予想欄**・独立した個人サイトに載る個人名分析（例：LIMO の個人寄稿）。参考に読むのは自由だが**要因の根拠にはせず**、外したものは `sources_excluded` に記す。
5. **裏取り規律**：検索結果の要約をそのまま出典にしない。**具体的な記事本文と配信時刻**を確認する。**株価・会社情報のランディングページ（Yahoo!ファイナンス `/quote`、日経 `/nkd/company`、株探の銘柄トップ等）は記事ではないため出典に挙げない**——見出しと配信時刻のある具体的な記事 URL のみを `sources_used`/`sources_new_candidate` に記す。ペイウォール等で取れない場合は他の①②媒体で裏取りし、できなければ「材料未確認」とする。
6. **正直性**：①②の範囲で窓に整合する材料が確認できなければ、創作せず **「材料未確認」**（または「当日固有の材料は確認できず」）と記す。
7. **文体**：要因文は `である調`・簡潔（1〜2文）。**数値は背景ドライバーに結びつける**（「上方修正」で止めず、受注・売価・需給・テーマまで踏み込む）。投資助言はしない。

## 出力形式

**(1) リサーチ本文（任意・推奨）**：エグゼクティブサマリー／株価・出来高の事実／上昇要因の多層分析／タイムライン／出典リンク等、自由な構成で根拠を示してよい。

**(2) DIGEST_BLOCK（必須・ファイル末尾に1個）**：Claude が機械的に取り込む。下記キーを過不足なく、この見出しサンチネルと key: value 形式で出力する。

```
## ダイジェスト用ブロック（DIGEST_BLOCK）
code: {{CODE}}
name: {{NAME}}
session_date: {{SESSION_DATE}}
区分: [開示] | [報道] | [テーマ]（複合可。例 [開示]＋[報道]）
trigger_time: YYYY-MM-DD HH:MM JST（直接トリガーの配信/開示時刻。不明なら「不明」）
window_ok: true | false | 該当なし（前営業日15:30以降∪当日15:30未満に整合するか）
要因: <である調・1〜2文。なぜ日中に上昇したか。数値は背景ドライバーに結びつける。材料未確認なら正直に>
sources_used:
  - 媒体: <①中核whitelist/一次情報> / 種別: <開示|報道> / 時刻: <YYYY-MM-DD HH:MM> / url: <URL>
sources_new_candidate:
  - 媒体: <②採用した非whitelist媒体> / 採用理由: <ルーブリック根拠> / url: <URL>
sources_excluded: <③として根拠から外したもの。無ければ「なし」>
broker_rating: <株探レーティング日報の確認結果。「該当なし」or「○○証券 中立→買い、目標株価X→Y、HH:MM配信」>
材料確度: 高 | 中 | 低 | 材料未確認
```

注：`sources_new_candidate`・`sources_excluded` が無ければ「なし」と書く。`区分` は主区分を1つ（必要なら複合）。配信時刻は分かる範囲で具体的に。

===== PROMPT END =====
